事件因果关系抽取现有的方法

2021-08-08- 2021-08-11
笔记

事件因果关系抽取现有的方法，以及其优势和劣势（包括传统基于句法的方法、基于规则的方法以及基于序列标注模型的方法）

第一阶段

基于句法和基于规则的方法只是有利于提取一因一果的句子，对于多因多果的提取方法较差。

  • 因果关系自动抽取的研究起源于20世纪90年代，早期研究围绕基于规则的方法展开。Grishman (1990 )等人提出了PROTEUS工具，使用语法和语义信息自动抽取文本中的时序关系和因果关系。

  • Kaplan（卡普兰）等(1991)将文本表示为命题的集合，每个命题包含一个谓词（通常为动词）和多个论元，通过定义命题模板的方式抽取命题中的因果关系。

  • Garcia （加西亚）（1997）提出了COATIS工具，该工具使用包含23个因果性动词的语言学模板自动从法语文本中抽取因果关系。

  • Chan.K等人(2005 ) U3]提出了基于语义期望的因果抽取系统SEKE，该系统使用了语义模板、句子模板、原因模板、后果模板四种不同层次的模板信息。

  • Girju ( 2002 ) 等人提出使用WordNet信息帮助自动发现表达因果含义的动词，用于抽取“NP 1-Verb-NP2”形式的因果关系，其中NP为名词短语，Verb为动词。

第二阶段

第二阶段主要使用基于统计与机器学习的方法。
缺点：准确率低，训练速度慢，需要大量数据集，并没有结合句子语义。

  • Girju（2003）提出了使用C4. 5决策树判断“NP 1-Verb-NP2”元组是否构成因果关系。

  • Chang等人（2004）使用无监督的方法从文本中学习短语指示因果关系的概率与词对出现在因果关系中的概率，作为额外特征训练贝叶斯分类器判断因果关系是否成立。

  • Blanco等人（2008）使用C4.5决策树结合Bagging方法进行因果关系识别。2007年，SemEval Task 4中提出了其中常见语义关系分类任务，其中包含因果关系，Girju等人使用SVM在该任务上取得了当时最佳的效果。

  • Sil等人（2010）提出了PREPOST系统，使用基于RBF核的SVM结合PMI特征判断因果关系是否成立。

  • 付剑锋等人(2011)提出使用层叠条件随机场的方法，在事件序列上同时标注多个因果关系对。

  • Zhao等人（2016）提出基于因果连接词类别信息的受限隐朴素贝叶斯模型提升因果抽取任务的效果。

  • Tharini等人（2017）将CNN应用于因果关系抽取任务中。

第三阶段

  • Kruengkrai等人（2017）进一步使用Multi-Column CNN（多柱卷积神经网络）并结合外部知识进行因果关系抽取。这一系列的研究将因果关系抽取建模为分类任务，输入是已经抽取好的事件或实体及其上下文，判断它们是否构成因果关系，而并未关注原因、结果事件或实体本身的抽取工作。

  • 随着自然语言处理技术的发展，一些研究者开始尝试使用序列标注模型解决因果关系抽取问题。

  • Dasgupta等人（2018）提出将因果关系抽取建模为序列标注任务，在文本中直接标注出原因提及与结果提及的短语。该工作使用双向LSTM作为序列标注模型，并加入额外的语言学特征提升该任务上的效果。

  • Dunietz等人（2018）提出了DeepCx模型用于浅层语义分析，并在因果关系抽取任务上进行了测试。

  • Li等人（2019）提出使用带有自注意力机制的双向LSTM-CRF模型，结合经过领域迁移的词向量应用于因果抽取任务。基于序列标注模型的因果抽取方法是一种端到端的方法，可以直接由文本得到因果事件或实体对，为因果关系抽取研究指明了新的方向。
